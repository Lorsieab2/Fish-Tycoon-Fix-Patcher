Fish Tycoon Fix Patcher - Transparency Log
==========================================

Release: v1.1.0
Date: 2026-07-21

Purpose
-------
Provide two independently selectable changes for the exact supported classic
Windows Fish Tycoon executable:

1. Repair the Crimson Comet curing branch with one 0..99 roll and success on
   0..19, followed by the game's existing clear and counter instructions.
2. Change the Unknown Chemical from one use to three uses and update its English
   store description to say "Contains 3 doses."

Crimson Comet evidence and implementation
------------------------------------------
The supported executable checks a nonzero disease/status field and calls
selector 2, whose target is body 11 and fin 11. It then calls sub_403240 with
zero. That helper can return only zero for this argument, so the following zero
test skips the original clear block.

The fix redirects that call to a 14-byte wrapper in verified alignment padding.
It calls the original helper once with 100 and returns one for 0..19 or zero for
20..99. The original identity check, test/jump, disease clear, and cure-counter
increment remain unchanged.

Unknown Chemical evidence and implementation
--------------------------------------------
The purchase branch sets the Unknown Chemical counter at state+0x2F8 to one.
The fix changes the immediate initializer in that full instruction from one to
three. The existing after-use decrement, zero test, and empty behavior remain
unchanged.

The unique English store-description substring "Contains one dose." is replaced
by the equal-size null-terminated string "Contains 3 doses." The separate German
localization text is not changed.

Distribution
------------
The repository and release contain source, a transparent manifest,
documentation, and tests. They contain no Fish Tycoon.exe or original game
assets. The patcher requires the exact supported executable and creates a
separate output folder by default.

Interpretation boundary
-----------------------
The byte-level behavior and observed runtime results are documented exactly.
The classification of the original Crimson Comet behavior as a developer bug
is the player's interpretation.
